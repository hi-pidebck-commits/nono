#### 2. 🛠️ 데이터 확보 방법론 (The "How")

API 연동이 최우선이지만, API가 제공하지 못하는 영역(질적 데이터, 맥락적 데이터)을 보강할 수 있는 방법을 제시합니다.

**A. 🔑 YouTube Data API 활용 가이드라인 (기술적 보강)**

*   **최우선 Scope 설정:** 단순히 `statistics`만 요청하지 말고, `search` (키워드 검색량, 유사 영상 검색), `comment` (질적 데이터 수집), `playlist` (주요 시리즈물 패턴 파악) 스코프를 요청할 수 있는지 검토해야 합니다.
*   **Rate Limit 관리:** API 호출 제한(Rate Limit)에 걸리지 않도록, 필요한 데이터를 **배치(Batch)** 단위로 요청하는 로직을 설계해야 합니다.
*   **데이터 전처리 프로세스:** API에서 받은 원본 JSON 데이터를 **'Pain Point'**와 **'CTA 키워드'**라는 두 가지 기준으로 필터링하고 분류하는 전처리 스크립트가 필요합니다.

**B. 🌐 경쟁사 데이터 수집 방법론 (Semi-Manual/Scraping)**

*   **키워드 기반 수동 분석 (Pain Point 매트릭스):**
    1.  특정 핵심 키워드 (예: 1인 CEO 자동화)를 선정합니다.
    2.  유튜브 검색창에서 해당 키워드를 검색하고, 상위 10개 영상을 추출합니다.
    3.  이 10개 영상의 제목, 설명, 그리고 **최상위 댓글**을 수동으로 캡처합니다.
    4.  이 데이터를 매트릭스에 정리하여, **'어떤 Pain Point가 가장 자주 언급되는지'**를 시각적으로 파악합니다. (API로 대체하기 어려우나, 가장 중요한 질적 데이터)

**C. 📈 시장 트렌드 교차 분석 (Keyword & Trend)**

*   **구글 트렌드 활용:** 'AI + 1인 기업', '자동화 + SaaS' 등 2~3개 키워드를 조합하여, 시간 경과에 따른 검색량 변화 추이를 확인합니다. 트렌드가 **급격히 상승하는(Rising)** 지점을 포착하는 것이 가장 중요합니다.
*   **커뮤니티 데이터 활용:** 네이버 지식인, 레딧(Reddit) 등 유튜브 외부의 커뮤니티에서 '1인 기업 자동화'에 대한 질문을 검색하고, **실제 사람들이 어떤 방식으로 해결책을 찾고 있는지** 그 맥락(Context)을 파악합니다.
